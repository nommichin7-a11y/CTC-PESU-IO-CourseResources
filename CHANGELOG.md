## Version History Legend

- **[Unreleased]**: Changes in development but not yet released
- **[Major.Minor.Patch]**: Version numbers following semantic versioning
  - **Major**: Significant course restructuring or breaking changes
  - **Minor**: New features, additional content, or enhancements
  - **Patch**: Bug fixes, typos, minor improvements

## Change Categories

- **Added**: New features, content, or capabilities
- **Changed**: Changes to existing functionality or content
- **Deprecated**: Features or content to be removed in future versions
- **Removed**: Features or content that have been deleted
- **Fixed**: Bug fixes or corrections
- **Security**: Security-related changes or improvements

## How to Contribute to Changelog

When submitting a pull request, please update this changelog:

1. Add your changes under **[Unreleased]**
2. Use appropriate category (Added, Changed, Fixed, etc.)
3. Write clear, concise descriptions
4. Link to relevant issues or PRs

Example:

```markdown
### Added

- New AES implementation notebook (#123)
- Interactive key schedule visualization

### Fixed

- Typo in Caesar cipher documentation (#124)
- Bug in RSA key generation for large primes (#125)
```
